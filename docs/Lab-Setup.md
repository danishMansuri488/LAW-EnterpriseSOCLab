
# Lab Setup

## Purpose

The laboratory was designed to simulate an enterprise Security Operations Center (SOC) environment for developing and validating detection engineering, security monitoring, threat hunting, and incident investigation workflows.

---

## Lab Environment

| Component | Technology |
|----------|------------|
| Host Operating System | Windows 11 |
| Hypervisor | Oracle VirtualBox |
| Domain Controller | Windows Server 2022 |
| Client Machine | Windows 11 |
| Identity Platform | Active Directory Domain Services (AD DS) |
| SIEM Platforms | Microsoft Sentinel, Splunk Enterprise |
| Endpoint Telemetry | Sysmon |
| Log Source | Windows Security Events |
| Cloud Platform | Microsoft Azure |

---

## Infrastructure

The laboratory consists of a Windows Server acting as the Domain Controller and a Windows client joined to the Active Directory domain. Security telemetry generated by both systems is collected and analyzed using Microsoft Sentinel and Splunk Enterprise.

---

## Log Sources

The following telemetry sources are used throughout the project:

- Windows Security Event Logs
- Sysmon Operational Logs
- Active Directory Authentication Events
- User Account Management Events
- Privileged Logon Events

---

## Monitoring Objectives

The laboratory focuses on monitoring:

- User authentication activity
- Failed and successful logons
- Account lockouts
- Privileged account usage
- User and group management
- Endpoint process activity
- Windows security events

---

## Validation Approach

Detection logic and investigation workflows were validated by generating controlled security events within the isolated laboratory environment. Generated telemetry was verified in both Microsoft Sentinel and Splunk Enterprise before documentation.
