
# Contributing

Thank you for your interest in contributing to LAW-EnterpriseSOCLab.

## Contribution Guidelines

Contributions that improve the quality, accuracy, or usability of this repository are welcome.

Examples include:

- Documentation improvements
- Detection engineering enhancements
- Threat hunting use cases
- KQL and SPL query optimizations
- Architecture improvements
- Bug fixes
- Typographical corrections

---

## Pull Requests

When submitting a pull request, please ensure:

- Documentation is clear and technically accurate.
- Markdown formatting is consistent.
- Detection logic is validated before submission.
- New content aligns with the project's scope.

---

## Issues

If you encounter a problem or have a suggestion, please open a GitHub Issue with:

- A clear description
- Expected behavior
- Suggested improvement (if applicable)

---

## Code of Conduct

Please maintain a professional and respectful environment when contributing to this project.
