# Incident Response

## Overview

This laboratory incorporates a structured incident response workflow to investigate and validate security alerts generated by Microsoft Sentinel and Splunk Enterprise. The process follows a repeatable methodology that supports efficient triage, analysis, and documentation of security events within an enterprise environment.

---

## Objectives

- Validate security alerts
- Determine the scope of suspicious activity
- Investigate affected users and systems
- Identify indicators of compromise (IOCs)
- Document investigation findings
- Support containment and remediation decisions

---

## Incident Response Workflow

```text
Alert Generated
       │
       ▼

       ▼

       ▼

       │
       ▼

       │
       ▼
Containment & Remediation
       │
       ▼
Documentation & Lessons Learned
```

---

## Investigation Process

Each investigation follows a structured workflow:

1. Review the generated alert.
2. Validate the triggering event.
3. Correlate related security events.
4. Analyze user and host activity.
5. Determine the root cause.
6. Document findings and recommendations.

---

## Investigation Data Sources

- Microsoft Sentinel
- Splunk Enterprise
- Windows Security Events
- Sysmon Operational Logs
- Active Directory
- Azure Log Analytics Workspace

---

## Investigation Scenarios

The laboratory includes investigations for:

- Failed Authentication Attempts
- Brute Force Activity
- Account Lockouts
- Privileged Logons
- New User Account Creation
- Group Membership Changes
- Remote Desktop Activity
- Suspicious Process Execution

---

## Documentation

Each validated investigation includes:

- Alert Summary
- Timeline of Events
- Evidence Collected
- Root Cause Analysis
- Findings
- Recommended Actions
